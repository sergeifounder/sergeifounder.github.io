7. Entity Graph — как AI связывает всё в одного человека

AI строит "карту знаний" по имени. Чем больше связей — тем увереннее AI что ты реальный эксперт.

ПлатформаИмяСвязь с проектами

ВАЖНО: одинаковое имя ВЕЗДЕ. Не "Sergei K." и не "S. Krushinskikh". AI связывает entity по точному совпадению.

Что AI видит когда кто-то спросит "Who built geoseo.today?":

1. geoseo.today → "Built by Sergei Krushinskikh" ✓
2. GitHub sergeifounder → "Author of geoseo-today" ✓
3. LinkedIn → "Founder @ IndoGig, Building geoseo.today" ✓
4. dev.to → статья "How I built GEO engine" ✓
5. Product Hunt → "Maker: Sergei Krushinskikh" ✓

5 независимых источников = AI уверен что Sergei реальный → цитирует с доверием

8. Schema JSON-LD — как связать проекты технически

На каждом сайте должен быть Person schema:

{
  "@context": "https://schema.org",
  "@type": "Person",
  "name": "Sergei Krushinskikh",
  "url": "https://sergeifounder.com",
  "sameAs": [
    "https://github.com/sergeifounder",
    "https://linkedin.com/in/sergei-krushinskikh",
    "https://twitter.com/sergeifounder",
    "https://x.com/sergeifounder"
  ],
  "jobTitle": "Founder",
  "worksFor": [
    {"@type": "Organization", "name": "IndoGig", "url": "https://indogig.com"},
    {"@type": "Organization", "name": "geoseo.today", "url": "https://geoseo.today"}
  ],
  "knowsAbout": ["SEO", "GEO", "AI Search Optimization", "Staffing", "Web Development"]
}

Этот JSON-LD вставить на: geoseo.today, indogig.com, sergeifounder.com, rosspaintinggroup.com (в footer/about)
